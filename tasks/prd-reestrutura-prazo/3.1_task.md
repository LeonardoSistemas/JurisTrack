# 3.1_task - Ajustar Frontend (Ficha do Processo e Dashboard)

## Objetivo
- Atualizar a interface do usuário para refletir as mudanças estruturais e permitir a gestão de status dos prazos.

## Escopo / Entregáveis
- Alterações em `public/js/fichaProcesso.js` e componentes de UI relacionados.

## Passos e subtarefas
- [ ] 3.1.1: Em `renderizarPrazos`: Ajustar a leitura dos dados para buscar prazos na raiz do objeto do processo.
- [ ] 3.1.2: Atualizar o template de exibição do prazo para mostrar a `descricao` e o `status` formatado.
- [ ] 3.1.3: Em `salvarPrazoManual`: Ajustar o envio do formulário para passar o `processoId` diretamente.
- [ ] 3.1.4: Adicionar elementos de UI (botões ou dropdown) para alteração de status (Concluir, Cancelar, etc.).

## Dependências
- [x] Task 2.1 e 2.3 (APIs atualizadas).

## Paralelizável?
- Sim, após a definição dos contratos de API.

## Critérios de aceite
- Prazos aparecendo corretamente na ficha do processo.
- Usuário consegue criar um prazo manual e ele aparece imediatamente na lista.
- Mudança de status refletida visualmente sem necessidade de recarregar a página (se possível).

## Testes
- Teste manual de fluxo de ponta a ponta: Criar prazo -> Ver na lista -> Alterar status -> Verificar persistência.
- Validar responsividade dos novos elementos de UI.

## Notas
- Manter a consistência visual com o restante do sistema.
