# 1.1_task - Criar Status e Auditoria no DB

## Objetivo
- Preparar a infraestrutura de dados para suportar o novo sistema de prazos, incluindo os estados possíveis e a rastreabilidade de mudanças.

## Escopo / Entregáveis
- Script SQL para inserção de status no `aux_status`.
- Script SQL para criação da tabela `prazo_auditoria`.

## Passos e subtarefas
- [ ] 1.1.1: Criar script SQL para inserir os status `pendente`, `concluido`, `cancelado`, `em atraso`, `suspenso` no domínio `prazo` da tabela `aux_status`.
- [ ] 1.1.2: Criar script DDL para a tabela `prazo_auditoria` com os campos: `id`, `prazo_id`, `status_anterior_id`, `status_novo_id`, `usuario_id`, `data_alteracao`.
- [ ] 1.1.3: Validar a criação das tabelas e inserção dos dados no ambiente de desenvolvimento.

## Dependências
- Nenhuma (Esta é a tarefa base).

## Paralelizável?
- Sim, pode ser executada em paralelo com a preparação dos serviços de backend (Task 2.x), desde que não dependam da execução imediata no banco.

## Critérios de aceite
- Tabela `aux_status` deve conter os novos registros para o domínio `prazo`.
- Tabela `prazo_auditoria` deve estar criada e pronta para receber registros.

## Testes
- Executar query de seleção em `aux_status` filtrando por domínio `prazo`.
- Verificar estrutura da tabela `prazo_auditoria` via ferramenta de banco de dados.

## Notas
- Seguir o padrão de nomes e tipos de dados já existentes no projeto.
