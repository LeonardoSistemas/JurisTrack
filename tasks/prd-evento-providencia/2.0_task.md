# 2.0 - Serviço de Motor de Regras e Prazos

## Objetivo
- Implementar a lógica de negócio (backend) que identifica eventos a partir de descrições brutas e calcula as sugestões de providências e prazos.

## Escopo / Entregáveis
- `src/services/eventoProvidenciaService.js`: Motor de match e busca de regras.
- `src/services/prazoService.js`: Lógica de cálculo de datas de vencimento.

## Passos e subtarefas
- 2.1 Implementar lógica de busca em `andamento_evento` (primeiro match exato, depois contém).
- 2.2 Criar função para retornar a providência padrão e alternativas para um evento.
- 2.3 Implementar cálculo de prazos úteis em `prazoService.js` (usando `date-fns` ou similar).
- 2.4 Integrar busca de modelos de petição vinculados à providência.
- 2.5 Criar tratamento para `EVENTO_NAO_CLASSIFICADO`.

## Dependências
- 1.0 (Infraestrutura de Banco de Dados) para consultas.

## Paralelizável?
- Não, depende da estrutura do banco de dados estar definida.

## Critérios de aceite
- O serviço retorna o `evento_id` correto para strings de teste variadas.
- O cálculo de 15 dias úteis a partir de uma data ignora fins de semana.
- Retorno estruturado conforme definido no contrato da API.

## Testes
- Testes unitários para o motor de match com diferentes entradas de texto.
- Testes unitários para o serviço de prazo validando feriados nacionais básicos.

## Notas
- Garantir que todas as buscas no banco incluam o filtro de `tenant_id`.
