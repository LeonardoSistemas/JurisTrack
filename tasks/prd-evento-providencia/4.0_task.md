# 4.0 - Modal de Análise e Lógica Frontend

## Objetivo
- Criar a interface de revisão (Modal) que permite ao advogado validar e ajustar as sugestões do sistema.

## Escopo / Entregáveis
- Componente de Modal no HTML/JS.
- `public/js/analisePublicacao.js` (Lógica de controle do modal).

## Passos e subtarefas
- 4.1 Desenvolver estrutura HTML do modal (Título, Texto da Publicação, Campos de Edição).
- 4.2 Implementar função `abrirModalAnalise(idItem)` que consome a API de sugestões.
- 4.3 Criar lógica de atualização dinâmica (ex: mudar providência recalcula prazo e modelo sugerido no modal).
- 4.4 Implementar botões de "Salvar" e "Salvar e Próximo".

## Dependências
- 3.0 (Endpoints da API) para carregamento e envio de dados.

## Paralelizável?
- Sim, pode-se iniciar o layout com dados mockados.

## Critérios de aceite
- O modal exibe o texto integral da publicação corretamente.
- Alterar o select de Providência reflete imediatamente nos campos de Prazo e Modelo.
- O modal é responsivo e não quebra o layout da página pai.

## Testes
- Testar interação do usuário (cliques, trocas de select, input de data).
- Validar comportamento em telas menores.

## Notas
- Usar bibliotecas já existentes no projeto para o Modal (ex: Bootstrap ou CSS customizado do sistema).
