# 1.0 - Infraestrutura de Banco de Dados

## Objetivo
- Criar a estrutura de dados necessária no Supabase para suportar o motor de regras, mapeamentos e auditoria de sugestões.

## Escopo / Entregáveis
- Migrações SQL para criação das tabelas.
- Script de Seed com regras iniciais para validação.

## Passos e subtarefas
- 1.1 Criar tabelas base: `evento_processual`, `andamento_evento`, `providencia_juridica`.
- 1.2 Criar tabelas de relacionamento e templates: `evento_providencia`, `modelo_peticao`.
- 1.3 Criar tabela de log: `auditoria_sugestao`.
- 1.4 Desenvolver script de seed com pelo menos 5 mapeamentos comuns (ex: Intimação, Sentença).

## Dependências
- Acesso ao console ou CLI do Supabase.

## Paralelizável?
- Sim, pode ser iniciada independente de outras tarefas de código.

## Critérios de aceite
- Todas as tabelas criadas no esquema `public`.
- Seed executado com sucesso e dados visíveis no dashboard.

## Testes
- Validar inserção e consulta manual via painel do Supabase.
- Garantir que todas as tabelas possuem a coluna `tenant_id` para controle via backend.

## Notas
- Seguir o padrão `snake_case` definido na Tech Spec.
