# 6.0 - Configuração de Regras (Evento -> Providência)

## Objetivo
- Criar a lógica e interface que define qual providência deve ser sugerida para cada evento, incluindo regras de prazo.

## Escopo / Entregáveis
- Endpoints para a tabela `evento_providencia`.
- Interface de configuração de regras (vínculo entre as duas entidades).

## Passos e subtarefas
- 6.1 Implementar CRUD de `evento_providencia` no backend.
- 6.2 Criar formulário de configuração com campos: Prioridade, Gera Prazo (bool), Dias, Tipo de Prazo (Útil/Corrido), Providência Padrão (bool).
- 6.3 Implementar lógica para garantir que apenas uma providência seja a "Padrão" por evento/tenant (opcional, mas recomendado).
- 6.4 Integrar interface na tela de Eventos ou Providências.

## Dependências
- Tarefa 3.0 (Eventos) e Tarefa 5.0 (Providências).

## Paralelizável?
- Não, depende de ambas as entidades estarem funcionais.

## Critérios de aceite
- É possível configurar que o evento "SENTENÇA" gera "INTERPOR RECURSO" em 15 dias úteis.
- As regras são salvas e recuperadas corretamente por tenant.

## Testes
- 6.5 Validar cálculos de prazo (dias úteis vs corridos) se houver helper disponível.

## Notas
- Esta é a "inteligência" do mapeamento jurídico.
