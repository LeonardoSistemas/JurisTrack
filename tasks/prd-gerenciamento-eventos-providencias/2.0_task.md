# 2.0 - Estrutura de Backend (Boilerplate)

## Objetivo
- Criar a estrutura básica de arquivos (rotas, controllers e services) para os módulos de Eventos e Providências.

## Escopo / Entregáveis
- Arquivos de rotas: `configuracaoEventoRoute.js`, `configuracaoProvidenciaRoute.js`.
- Arquivos de controllers: `configuracaoEventoController.js`, `configuracaoProvidenciaController.js`.
- Arquivos de services: `configuracaoEventoService.js`, `configuracaoProvidenciaService.js`.

## Passos e subtarefas
- 2.1 Criar `src/routes/configuracaoEventoRoute.js` e registrar no `app.js` (ou arquivo central de rotas).
- 2.2 Criar `src/routes/configuracaoProvidenciaRoute.js` e registrar no `app.js`.
- 2.3 Criar controllers vazios com a estrutura de try/catch padrão do projeto.
- 2.4 Criar services vazios injetando as dependências de banco de dados.
- 2.5 Garantir que todas as rotas utilizem o middleware `ensureTenantAuthorization`.

## Dependências
- Estrutura base do projeto Node.js/Express.

## Paralelizável?
- Sim, com a tarefa 1.0.

## Critérios de aceite
- Servidor inicia sem erros com as novas rotas registradas.
- Estrutura de arquivos segue o padrão MVC do projeto.

## Testes
- 2.6 Realizar uma chamada GET básica para os novos endpoints e receber 200 OK (mesmo que com array vazio).

## Notas
- Manter o desacoplamento criando novos arquivos em vez de inflar os existentes.
