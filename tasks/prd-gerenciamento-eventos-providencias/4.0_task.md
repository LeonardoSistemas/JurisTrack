# 4.0 - Mapeamento de Andamentos

## Objetivo
- Permitir que o administrador mapeie strings de andamentos reais dos tribunais para os eventos internos do sistema.

## Escopo / Entregáveis
- Endpoints para `andamento_evento`.
- Seção de mapeamento na tela de Eventos ou modal específico.

## Passos e subtarefas
- 4.1 Implementar `GET` e `POST` para `andamento_evento` no service e controller de eventos.
- 4.2 Criar interface de listagem de mapeamentos (ex: "Designada Audiência de Instrução" -> "AUDIÊNCIA").
- 4.3 Implementar funcionalidade de exclusão de mapeamento.
- 4.4 Garantir que a busca de eventos para o mapeamento liste apenas eventos do tenant.

## Dependências
- Tarefa 3.0 (API e Interface de Eventos).

## Paralelizável?
- Sim, com tarefas de Providências (5.0, 7.0).

## Critérios de aceite
- É possível associar uma frase de tribunal a um evento interno.
- A lista de mapeamentos é exibida corretamente.

## Testes
- 4.5 Validar se o mapeamento funciona corretamente ao simular a entrada de uma string de tribunal.

## Notas
- Esta tarefa é crucial para a automação de leitura de andamentos.
