# 5.0 - API e Interface de Providências

## Objetivo
- Implementar o gerenciamento de providências jurídicas (ações a serem tomadas).

## Escopo / Entregáveis
- Endpoints CRUD para `providencia_juridica`.
- Tela administrativa `public/html/providencias.html`.
- Script de controle `public/js/providencias.js`.

## Passos e subtarefas
- 5.1 Implementar `GET`, `POST`, `PUT` em `configuracaoProvidenciaService.js` para a tabela `providencia_juridica`.
- 5.2 Implementar métodos correspondentes no `configuracaoProvidenciaController.js`.
- 5.3 Criar interface em `public/html/providencias.html` usando Bootstrap.
- 5.4 Desenvolver `public/js/providencias.js` para consumir a API.

## Dependências
- Tarefa 2.0 (Estrutura de Backend).

## Paralelizável?
- Sim, com a tarefa 3.0.

## Critérios de aceite
- Usuário consegue cadastrar uma providência (ex: "Interpor Agravo").
- Listagem funcional com paginação ou scroll infinito se necessário.

## Testes
- 5.5 Validar isolamento por tenant na criação e listagem.

## Notas
- A tela de providências será o hub para configurar checklists e modelos posteriormente.
