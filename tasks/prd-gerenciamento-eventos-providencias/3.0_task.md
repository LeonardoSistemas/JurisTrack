# 3.0 - API e Interface de Eventos

## Objetivo
- Implementar o gerenciamento completo de tipos de eventos processuais.

## Escopo / Entregáveis
- Endpoints CRUD para `evento_processual`.
- Tela administrativa `public/html/eventos.html`.
- Script de controle `public/js/eventos.js`.

## Passos e subtarefas
- 3.1 Implementar `GET`, `POST`, `PUT` em `configuracaoEventoService.js` para a tabela `evento_processual`.
- 3.2 Implementar métodos correspondentes no `configuracaoEventoController.js`.
- 3.3 Criar interface em `public/html/eventos.html` usando Bootstrap (Tabela de listagem e Modal de cadastro/edição).
- 3.4 Desenvolver `public/js/eventos.js` para consumir a API e manipular o DOM.
- 3.5 Adicionar filtros de busca e status (Ativo/Inativo) na interface.

## Dependências
- Tarefa 2.0 (Estrutura de Backend).

## Paralelizável?
- Sim, pode ser feito em paralelo com a tarefa 5.0 (Providências).

## Critérios de aceite
- Usuário consegue cadastrar um novo evento (ex: "AUDIÊNCIA").
- Usuário consegue editar o nome ou status de um evento existente.
- A listagem reflete apenas os dados do tenant logado.

## Testes

- 3.7 Validar se campos obrigatórios impedem o envio do formulário.
