# 3.0 - Ajuste no Fluxo de Cadastro (Backend)

## Objetivo
- Alterar o comportamento do botão "Cadastrar" para que a publicação seja gerada sem criar o registro de prazo imediatamente.

## Escopo / Entregáveis
- Modificação no serviço/controller de similaridade:
    - Remoção da chamada ao repositório de prazos.
    - Inclusão da lógica de persistência na tabela de vínculo.
    - Atualização do status do item.

## Passos e subtarefas
- 3.1 Localizar e remover a lógica que cria o registro na tabela `Prazo` durante o cadastro de similaridade.
- 3.2 Implementar a chamada ao `similaridadeRepository.vincularPublicacao` (ou equivalente) para salvar o vínculo na nova tabela.
- 3.3 Atualizar o `status_decisao` do item de similaridade para `cadastrado_sem_prazo`.
- 3.4 Garantir que todo o processo (Processo -> Publicação -> Vínculo -> Status) ocorra dentro de uma transação.

## Dependências
- 2.0 (Migração do Banco de Dados).

## Paralelizável?
- Não.

## Critérios de aceite
- Ao cadastrar um item de similaridade, os registros de Processo, Publicação e Andamento são criados, mas a tabela `Prazo` permanece sem novos registros para este item.
- O status do item muda para `cadastrado_sem_prazo`.

## Testes
- 3.5 Testar cadastro com sucesso e verificar ausência de prazo no banco.
- 3.6 Testar falha proposital no vínculo e garantir que a Publicação não seja persistida (rollback).

## Notas
- Manter o retorno do ID da publicação para o frontend após o cadastro.
