# 2.0 - Migração do Banco de Dados

## Objetivo
- Criar a estrutura necessária no banco de dados para suportar o vínculo entre itens de similaridade e publicações, além de permitir o rastreio da auditoria de IA nos prazos.

## Escopo / Entregáveis
- Script SQL de migração contendo:
    - Criação da tabela `similaridade_item_publicacao`.
    - Alteração da tabela `Prazo` (coluna `auditoria_sugestao_id`).
    - (Opcional) Registro ou atualização de ENUM/Constraint para novos status.

## Passos e subtarefas
- 2.1 Criar a tabela `similaridade_item_publicacao` com as colunas: `id`, `item_similaridade_id`, `publicacao_id`, `tenant_id`, `created_at`.
- 2.2 Adicionar a coluna `auditoria_sugestao_id` (FK) na tabela `Prazo`.
- 2.3 Verificar e atualizar as constraints de `status_decisao` na tabela `similaridade_itens` para aceitar `cadastrado_sem_prazo` e `analisado_com_prazo`.

## Dependências
- Nenhuma.

## Paralelizável?
- Não (Base para as tarefas de lógica de negócio).

## Critérios de aceite
- Tabela criada e colunas adicionadas sem erro no banco de desenvolvimento.
- Constraints de status validadas.

## Testes
- 2.4 Validar inserção manual de um registro na nova tabela de vínculo.
- 2.5 Verificar se a coluna nova em `Prazo` permite valores nulos (para prazos criados por outros fluxos).

## Notas
- Utilizar o padrão de UUID do projeto.
