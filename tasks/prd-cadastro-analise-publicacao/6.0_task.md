# 6.0 - Integração do Modal de Análise

## Objetivo
- Garantir que o modal de análise da IA receba os IDs corretos da publicação cadastrada para vincular o prazo final.

## Escopo / Entregáveis
- Atualização da lógica de abertura do modal no frontend.
- Passagem de parâmetros de contexto para o componente de análise.

## Passos e subtarefas
- 6.1 Modificar a função que abre o modal de análise para buscar o `publicacao_id` vinculado ao item de similaridade.
- 6.2 Enviar `item_similaridade_id` e `publicacao_id` no payload de fechamento/confirmação do modal.
- 6.3 Garantir que o modal exiba os dados da publicação correta para a análise do advogado.

## Dependências
- 4.0 (Endpoint de Confirmação).
- 5.0 (Habilitação do botão Analisar).

## Paralelizável?
- Não.

## Critérios de aceite
- Ao clicar em "Analisar", o modal abre com os dados da publicação cadastrada.
- Ao clicar em "Confirmar" no modal, os IDs corretos são enviados para o backend.

## Testes
- 6.4 Validar se o modal carrega as informações esperadas.
- 6.5 Verificar no console do navegador se os parâmetros estão sendo enviados corretamente na requisição final.

## Notas
- O `publicacao_id` pode precisar ser buscado na nova tabela de vínculo se não estiver presente no estado local do card.
