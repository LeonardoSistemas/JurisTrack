# 7.0 - Testes de Fluxo Ponta a Ponta

## Objetivo
- Validar o ciclo de vida completo da funcionalidade, garantindo que não existam regressões e que os dados estejam íntegros.

## Escopo / Entregáveis
- Roteiro de testes executado.
- Validação de banco de dados pós-fluxo.

## Passos e subtarefas
- 7.1 Cenário A: Cadastro de item pendente -> Verificar status `cadastrado_sem_prazo` e ausência de prazo.
- 7.2 Cenário B: Análise de item cadastrado -> Verificar criação de `Prazo` e transição para `analisado_com_prazo`.
- 7.3 Cenário C: Tentativa de cancelamento pós-cadastro -> Verificar bloqueio na UI.
- 7.4 Cenário D: Integridade de Dados -> Verificar se o `Prazo` criado no final do fluxo está vinculado à `Publicacao` correta e possui o `auditoria_sugestao_id`.

## Dependências
- Todas as tarefas anteriores (1.0 a 6.0).

## Paralelizável?
- Não.

## Critérios de aceite
- Fluxo completo realizado com sucesso em ambiente de desenvolvimento.
- Nenhuma falha de constraint ou inconsistência de dados detectada.

## Testes
- Re-execução de todos os testes descritos nas subtarefas de cada tarefa individual.

## Notas
- Testar também o caso de erro no meio do processo para garantir que o sistema não fique em estado inconsistente.
