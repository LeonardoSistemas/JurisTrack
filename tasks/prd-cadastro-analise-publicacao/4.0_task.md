# 4.0 - Evolução do Endpoint de Confirmação

## Objetivo
- Atualizar o endpoint que finaliza a análise da IA para criar o prazo processual definitivo e vincular à auditoria.

## Escopo / Entregáveis
- Atualização do endpoint `/api/analise/confirmar` no backend:
    - Recebimento de novos parâmetros (`publicacao_id`, `item_similaridade_id`).
    - Criação do registro em `Prazo`.
    - Atualização final do status do item.

## Passos e subtarefas
- 4.1 Modificar o DTO/Controller para receber `item_similaridade_id` e `publicacao_id`.
- 4.2 Integrar a criação do prazo com o `auditoria_sugestao_id` gerado pela análise.
- 4.3 Implementar a atualização do status do item para `analisado_com_prazo`.
- 4.4 Garantir atomicidade na criação do prazo e atualização do status.

## Dependências
- 2.0 (Migração do Banco de Dados).
- 3.0 (Para testar o fluxo completo).

## Paralelizável?
- Sim, com a tarefa 5.0.

## Critérios de aceite
- O endpoint cria um registro na tabela `Prazo` vinculado corretamente à Publicação e à Auditoria.
- O status do item de similaridade transita para `analisado_com_prazo`.

## Testes
- 4.5 Validar chamada da API com dados de análise e verificar persistência do prazo.
- 4.6 Verificar se o `auditoria_sugestao_id` foi salvo corretamente no Prazo.

## Notas
- O `prazo_final` deve vir do payload enviado pelo modal de análise.
